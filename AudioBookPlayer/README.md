# AudioBookPlayer

Плеер аудиокниг для Android: MP3 (главы = отдельные файлы), таймер сна, закладки.
Дизайн плеера — в духе Яндекс Книг (тёмная тема, крупная обложка, тонкий прогресс-бар,
жёлтый акцент, капсулы для скорости/таймера).

## Как открыть
1. Android Studio (Koala/Ladybug и новее) → **Open** → выбрать папку `AudioBookPlayer`.
2. Дать Gradle синхронизироваться (подтянет Media3, Room и остальное из `app/build.gradle.kts`).
3. Запустить конфигурацию `app` на эмуляторе/устройстве Android 7.0+.

## Что уже реализовано
- **Media3 `MediaSessionService`** (`player/PlaybackService.kt`) — воспроизведение,
  системное уведомление, экран блокировки, пауза при отключении наушников — всё "из коробки",
  без ручной сборки Notification, как в исходном плане.
- **Room + Flow**: `Book`, `Chapter`, `Bookmark` — экран библиотеки обновляется автоматически.
- **LibraryScanner** — оба способа добавления книг:
  - ручной выбор нескольких MP3-файлов (`GetMultipleContents`),
  - сканирование папки (`OpenDocumentTree`) — все аудиофайлы внутри становятся главами.
  Метаданные (название, автор, длительность) читаются через `MediaMetadataRetriever` —
  штатный класс Android SDK, никаких сомнительных сторонних либ.
- **SleepTimer** с поддержкой "продлить на N минут".
- **PlayerViewModel** подключается к сервису через `MediaController` — официальный способ
  Media3, не привязанный к жизненному циклу конкретной Activity.
- Закладки: добавление на текущей позиции, список через `Flow`.
- Экраны: `LibraryActivity` (список книг + добавление), `PlayerActivity` (плеер).

  можно добавлять поверх этой архитектуры без переделок.

## Почему не как в исходном плане
Кратко (подробности — в чате): устаревший `com.google.android.exoplayer` заменён на
актуальный `androidx.media3`; ручной `Foreground Service` — на `MediaSessionService`
с нормальным `MediaSession`; несуществующая библиотека метаданных — на штатный
`MediaMetadataRetriever`; списки в БД переведены на `Flow` для реактивного UI.
