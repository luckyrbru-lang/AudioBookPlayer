# AudioBookPlayer — контекст проекта для Claude Code

## Что это
Android-приложение для прослушивания аудиокниг: MP3 (главы = отдельные файлы),
таймер сна, закладки. Дизайн плеера — в духе Яндекс Книг (тёмная тема, крупная
квадратная обложка, тонкий жёлтый прогресс-бар, капсулы для скорости/таймера).

## Стек и архитектура
- Kotlin, MVVM, minSdk 24, compileSdk/targetSdk 34.
- **Media3** (`androidx.media3`) для воспроизведения — НЕ устаревший
  `com.google.android.exoplayer`. `player/PlaybackService.kt` — это
  `MediaSessionService`, который сам создаёт уведомление, отдаёт управление
  на экран блокировки/наушники. UI подключается к нему через `MediaController`,
  а не напрямую (сервис нельзя передать в конструктор ViewModel).
- **Room + Flow** — экран библиотеки обновляется реактивно, без ручного рефреша.
- Метаданные MP3 (название/автор/длительность) читаются через штатный
  `MediaMetadataRetriever` — без сторонних библиотек для тегов.
- Добавление книг — два способа (`data/scanner/LibraryScanner.kt`):
  ручной выбор файлов (`GetMultipleContents`) и сканирование папки
  (`OpenDocumentTree`), все аудиофайлы в папке = главы одной книги.
- **Без Hilt.** DI — вручную через `utils/ViewModelFactory.kt`. Так и оставляем,
  пока проект не вырастет настолько, что это станет проблемой.
- Книга = `Book` (метаданные) + список `Chapter` (по одному MP3-файлу на главу),
  связаны через `bookId`.

## Структура
```
app/src/main/java/com/example/audiobookplayer/
├── data/
│   ├── model/        Book, Chapter, Bookmark (Room entities)
│   ├── database/      AppDatabase + DAO (BookDao, ChapterDao, BookmarkDao)
│   ├── repository/    BookRepository — единственная точка входа в БД для ViewModel
│   └── scanner/        LibraryScanner — чтение файлов/папок и метаданных
├── player/
│   ├── PlaybackService.kt   MediaSessionService — ядро воспроизведения
│   └── SleepTimer.kt        таймер сна с "продлить на N минут"
├── ui/
│   ├── library/        LibraryActivity, BooksAdapter — список книг
│   └── player/         PlayerActivity, PlayerViewModel — экран плеера
└── utils/ViewModelFactory.kt
```

## Что уже готово (MVP)
- Плеер: play/pause, перемотка на 15/30 сек, слайдер прогресса, скорость воспроизведения.
- Таймер сна с пресетами (5/15/30/60 мин).
- Закладки: добавление на текущей позиции, хранение в БД.
- Библиотека: список книг, добавление файлами или папкой.

## Правила работы над проектом
- Не возвращать `com.google.android.exoplayer` — только `androidx.media3`.
- Не добавлять сторонние библиотеки для чтения тегов — `MediaMetadataRetriever` хватает.
- Списки из БД — только через `Flow`, не через `suspend fun getAll()`.
- Не подключать Hilt без явной просьбы.
- Дизайн — тёмная тема, стиль Яндекс Книг (минимализм, крупная обложка, тонкие
  элементы управления). Плюс покерная тема (владелец — покерист): приложение
  называется **All-In Audio**, акцент — не цвет, а тонкие чёрно-белые игральные
  масти (♠ ♣ ♦ ♥) как элемент управления/декор, а не цветастый вегас-стиль.
  Сейчас использовано: ♠ — закладки, ♣ — таймер сна, ♦ — скорость воспроизведения.
  Иконка приложения — `drawable/ic_launcher_poker.xml` (пиковая масть на чёрном).
  Splash-экран — `drawable/ic_splash_spade.xml` + `Theme.AudioBookPlayer.Splash`.
  Не скатывать это в кислотный "казино"-стиль — держать минимализм.
- Проект собирается через Gradle (`./gradlew assembleDebug`) — после правок
  желательно проверять сборку, если Android SDK доступен в окружении.
